我的hexo博客搭建

此文章为分享我在mac上搭建hexo博客的经历

基于b站代码羊CodeSheep的视频《手把手教你从0开始搭建自己的个人博客 |无坑版视频教程| hexo》

第一步：安装环境

1、进入Node.js官网安装Node.js

官网链接https://nodejs.org/en/，下载安装包，跟着安装包一步一步来就🉑️

通过在命令行（终端/terminal）中输入node -v查看版本号，以检查是否安装成功

2、利用npm安装淘宝镜像源

  • 在命令行输入npm install -g cnpm -registry-https://registry.npm.taobao.org 将自动安装镜像源

  • 在命令行输入cnpm -v查看版本号，检查是否安装成功

这样就可以使用cnpm工具，下载速度更快

3、安装hexo环境

在命令行输入cnpm install -g hexo-cli，安装hexo环境

4、新建博客文件夹

emmmmm新建文件夹大家都会吧，叭用我多说了吧

以我自己为例子啊

  • 在命令行，/user/liusihan目录下，输入mkdir myblog，新建名为”myblog”的文件夹，

此文件夹将用于存储博客文件，文件名可自定义嗷

后面的博客搭建出现什么问题，大不了整个文件夹删掉重来

5、使用hexo生成我们的博客

  • 在命令行使用”cd”命令跳转到我们刚刚建好的博客文件夹下

  • 输入sudo hexo init初始化一个博客

  • 初始化结束后，可以用”ls -l”命令，查看该文件夹下生成的文件

“_config.yml”是博客的配置文件，”/source/“是用于存放各个页面的文件夹，”/themes/“用于存放下载的主题

6、在本地查看搭建的博客

  • 在"user/liusihan/myblog/"目录下（即我们的博客文件夹目录下），在命令行输入hexo s命令（全称为hexo new）在本地运行博客

  • 输入命令后，可在浏览器地址栏输入”localhost:4000”，即可在4000端口从本地查看博客

  • 要结束运行，在命令行按下”control+c”即可（再次提醒，我用mac搭建的，可能和windows操作方法有点区别，大同小异）

第二步：将本地的博客部署到github/gitee上

1、注册/登陆到github/gitee（”码云”）

部署到github或者gitee上，搭建静态博客，建议可以的话，部署到gitee上，因为github慢，我懒得挂代理👀

因为摆烂，反正搭建了也没人看，就自己瞎写写东西，而且懒，并且没钱租服务器

以我为例子的话，我这里是部署到github上的，gitee的话，可以看看这篇文章Hexo 部署到 Gitee （面向CSDN编程，笑）

  • 新建一个仓库

    仓库名字一定是要”你的用户名.github.io”

    以我为例，我的用户名为”HalberdSL”，即我用于部署hexo博客的仓库名为”HalberdSL.github.io”（应该是不区分大小写的，反正我分了），Description就随便写

  • 在命令行博客文件夹的目录下，安装一个git部署的插件

    以我的为例，在命令行/user/liusihan/myblog目录下，输入命令"cnpm install --save hexo-deployer-git "

  • 配置”_config.yml”文件

    在命令行/user/liusihan/myblog目录下，输入vim _config.yml指令

    找到# Deployment这一块

    修改为：

    type: git

    添加：

    repo: git@github:HalberdSL/HalberdSl.github.io.git

    • 此处添加的是自己仓库的链接，建议使用ssh链接，在2021.8.13后，github的https链接好像不可以使用了，会报错）

    branch: master(部署在仓库的主分支)

  • 配置好后，在命令行myblog目录下输入sudo hexo cl && hexo g && hexo d将其部署到github上

    • hexo cl：即hexo clean，清理缓存
    • hexo g：即hexo generate，生成文件
    • hexo d：即hexo deploy，部署文件至github

    如果是用的https链接，可能会要求输入用户名和密码，如果是用的ssh则不需要

    可以在命令行进行配置

2、进行访问

部署成功后，在浏览器可以用仓库名进行访问，以我的为例”HalberdSL.github.io”

第三步：设置主题

客制化，懂得都懂

1、下载主题并配置

找到自己想要的hexo主题及其github/gitee仓库链接

在命令行用git clone github仓库链接 themes/主题名字 下载主题文件就行了

我用的是叫”ayer”的主题，ayer作者的博客，ayer的github链接，我这里就以”ayer”为例子

  • 在命令行用git clone https://github.com/Shen-Yu/hexo-theme-ayer themes/ayer

    下载主题文件至 myblog/themes/目录下

  • 在myblog文件夹下，使用命令vim _config.yml修改配置文件

    找到# Extensions这一块，将themes: landscape中的”landescape”替换为你使用的主题名字，我这里就是替换为”ayer”

  • 配置好后，在命令行myblog目录下输入sudo hexo cl && hexo g && hexo d将其部署到github上

    也可用hexo s在本地预览，修改后再部署到github上

2、设置页面

博客的具体配置就要看主题的介绍了，建议先到所使用的主题的作者的博客去看看

  • 在命令行输入hexo new page categories可以新建一个名为”categories”（分类）的页面

    标签（tags），友情链接（friends）等同理，取名看自己喜好

  • 新建好页面后，到/source/categories/index.md 文件（这里以”categories”举例，标签友链什么的同理）

    将以下复制到该文件中（vim index.md命令）

  • 在此文件内，就可以新建分类名，例如我就新建了一个”博客搭建经验”的分类

  • sudo hexo cl && hexo g && hexo d叭用我多说了吧

具体配置看各个主题不同嗷，不多谈

第四步：写文章

1、安装图片插件

写文章嘛，总会遇到想插两张表情包图片什么的时候

markdown语法中的![图片标题](图片.jpg)在这儿好像不起作用，具体请看官网的描述：资源文件夹

  • 在命令行输入cnpm install hexo-asset-image --ssave 安装插件

  • 打开博客文件夹/user/liusihan/myblog下的配置文件_config.yml

    将文件中的post_asset_folder选项改为true

    这样，在新建文章的时候，会同时创建一个和文章同名的文件夹，用于存放文章要添加的图片

2、新建文章

  • 在命令行中输入hexo n "文章标题"，即可在/myblog/source/_posts文件夹下新建为”文章标题.md”的markdown文件

    如果安装了hexo-asset-image插件的话，还会生成一个同名文件夹

    文章是markdown格式的，可以用markdown语法进行编辑

  • 可以通过vim 文章标题.md命令来编辑，或者用Typora打开文章进行编辑

3、添加图片

  • 要添加图片时，在文章中输入{% asset_img 想添加的图片.jpg 图片标题%}

    jpg/jpeg/png好像都🉑️

  • 用md语法添加图片![标题](image.jpg)来添加图片

    似乎不会显示在首页或归档中，但是能在文章中正确显示

    也可能首页、归档和文章中都无法加载
